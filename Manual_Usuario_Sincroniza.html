1. INTRODUCCIÓN

1.1 ¿Qué es Sincroniza?

Sincroniza es una aplicación para Microsoft Dynamics 365 Business Central que permite sincronizar datos entre diferentes empresas dentro del mismo entorno. La aplicación facilita la gestión de datos maestros y la replicación de información entre empresas filiales y la empresa principal.

1.2 Características Principales

  • Sincronización Automática: Los datos se sincronizan automáticamente cuando se realizan cambios
  • Configuración Flexible: Permite seleccionar qué tablas y campos sincronizar
  • Gestión de Empresa Maestra: Identifica y gestiona la empresa principal del sistema
  • Copia de Empresa: Facilita la creación de nuevas empresas con datos predefinidos
  • Interfaz Intuitiva: Páginas fáciles de usar para la configuración

1.3 Beneficios

  • Consistencia de Datos: Mantiene la información sincronizada entre empresas
  • Reducción de Errores: Elimina la necesidad de entrada manual duplicada
  • Eficiencia Operativa: Ahorra tiempo en la gestión de datos
  • Flexibilidad: Permite personalizar qué datos sincronizar

7. PROCESO DE SINCRONIZACIÓN

7.1 Funcionamiento Automático

La sincronización funciona automáticamente cuando:

  • Se Modifica un Registro: Cualquier cambio en una tabla configurada
  • Se Inserta un Registro: Nuevos registros en tablas configuradas
  • Se Elimina un Registro: Eliminación de registros en tablas configuradas
  • Se Renombra un Registro: Cambios de nombre en registros

7.2 Flujo de Sincronización

Cambio en Empresa A (Maestra) ↓ Detección Automática ↓ Verificación de Configuración ↓ Aplicación en Empresa B (Filial) ↓ Confirmación de Sincronización

7.3 Tipos de Sincronización

Tipo Descripción Comportamiento
Inserción Nuevos registros Se crean en empresas filiales manteniendo la integridad referencial
Modificación Cambios en registros existentes Se propagan actualizando información desactualizada
Eliminación Eliminación de registros Se replican en empresas filiales manteniendo consistencia
Renombrado Cambios de nombre Se sincronizan preservando referencias entre registros

8. COPIA DE EMPRESA

8.1 Funcionalidad de Copia

La función de copia de empresa permite:

  • Crear Nuevas Empresas: Con datos predefinidos
  • Replicar Configuraciones: Mantener consistencia entre empresas
  • Sincronizar Datos Maestros: Asegurar datos iniciales correctos

8.2 Proceso de Copia

Ubicación: Configuración > Sincronización > Copiar Empresa

8.2.1 Configuración de Copia

Campo Descripción Validación Uso
Nueva Empresa Nombre de la empresa destino Debe existir en el sistema Especifica dónde se copiarán los datos
Lista de Tablas Muestra todas las tablas disponibles N/A Selección de tablas a copiar
ID del Objeto Identificador de la tabla N/A Referencia interna
Leyenda del Objeto Nombre descriptivo de la tabla N/A Identificación visual
Campos Excluidos Configuración de campos a excluir N/A Control de campos específicos

8.2.2 Ejecución de Copia

  1. Seleccionar Empresa Destino: Especificar la empresa donde copiar
  2. Configurar Tablas: Marcar las tablas a copiar
  3. Configurar Campos: Especificar campos a excluir si es necesario
  4. Ejecutar Copia: Hacer clic en "Copiar Empresa"
  5. Verificar Resultado: Confirmar que la copia se completó correctamente

8.2.3 Consideraciones Importantes

Rendimiento

  • El proceso puede tomar tiempo dependiendo del volumen de datos
  • Se recomienda ejecutar durante horarios de bajo uso

Integridad de Datos

  • Verificar que la empresa destino esté correctamente configurada
  • Asegurar que no haya conflictos de datos

Permisos

  • El usuario debe tener permisos de escritura en la empresa destino
  • Verificar permisos de sincronización

9. SOLUCIÓN DE PROBLEMAS

9.1 Problemas Comunes

9.1.1 Sincronización No Funciona

Síntomas:

  • Los cambios no se reflejan en otras empresas
  • No aparecen mensajes de confirmación

Soluciones:

  1. Verificar que la empresa esté marcada como maestra
  2. Comprobar la configuración de tablas
  3. Revisar permisos de usuario
  4. Verificar que las empresas destino existan

9.1.2 Errores de Permisos

Síntomas:

  • Mensajes de acceso denegado
  • No se pueden realizar cambios en la configuración

Soluciones:

  1. Verificar que el usuario tenga el conjunto de permisos SINCRONIZA
  2. Contactar al administrador del sistema
  3. Revisar la configuración de seguridad

9.1.3 Datos Inconsistentes

Síntomas:

  • Información diferente entre empresas
  • Registros duplicados o faltantes

Soluciones:

  1. Revisar la configuración de campos excluidos
  2. Verificar que la sincronización esté activa
  3. Ejecutar una sincronización manual si es necesario

9.2 Logs y Monitoreo

Registro de Cambios

  • La aplicación mantiene un registro de todas las sincronizaciones
  • Accesible desde la configuración de tablas

Mensajes de Error

  • Los errores se registran en el log de aplicación
  • Revisar el visor de eventos para detalles

10. PREGUNTAS FRECUENTES

10.1 Configuración

P: ¿Puedo cambiar qué empresa es la maestra?

R: Sí, pero solo debe haber una empresa maestra a la vez. Cambiar esta configuración puede afectar el rendimiento.

P: ¿Cómo sé si una tabla está configurada para sincronizar?

R: En la lista de tablas de sincronización, la columna "Registrar Modificación" indica si la tabla está activa.

P: ¿Puedo excluir campos específicos de la sincronización?

R: Sí, en la configuración de campos puede marcar qué campos excluir.

10.2 Funcionamiento

P: ¿Cuándo se sincronizan los datos?

R: La sincronización es automática y ocurre inmediatamente cuando se realizan cambios en tablas configuradas.

P: ¿Qué pasa si hay un error durante la sincronización?

R: El error se registra en el log y la sincronización se detiene. Debe revisar y corregir el problema antes de continuar.

P: ¿Puedo deshacer una sincronización?

R: No, la sincronización es irreversible. Se recomienda hacer respaldos antes de cambios importantes.

10.3 Rendimiento

P: ¿La sincronización afecta el rendimiento del sistema?

R: Puede haber un impacto mínimo durante la sincronización, especialmente con grandes volúmenes de datos.

P: ¿Cuántas empresas puedo sincronizar?

R: No hay límite técnico, pero el rendimiento puede verse afectado con muchas empresas.
